# 项目简报：作品设定工具

## 产品判断

原始资料里有非常多宏大的 StoryOS、联邦治理、商业信任治理设计。它们不是没有价值，但对当前项目启动来说，最重要的是收束。

这个项目的第一性问题不是“如何设计一个完整 AI 创作操作系统”，而是：

> 一个作者输入一个模糊创意后，能不能在 10 到 30 分钟内得到一份结构清晰、可继续扩写、可被 AI 引用的作品设定资料？

如果这个问题能解决，项目就有真实价值。

## 目标用户

初期只服务三类用户：

1. 网文作者：需要快速搭建长篇设定、人物关系、卖点和连载方向。
2. 剧本 / 游戏叙事创作者：需要管理世界观、阵营、冲突和伏笔。
3. AI 辅助创作者：需要让 AI 持续记住项目上下文，而不是每次重新解释。

## 核心价值

作品设定工具的价值不在于“帮作者写完所有内容”，而在于：

- 把灵感变成结构。
- 把结构变成资产。
- 把资产变成可复用上下文。
- 把上下文变成持续协作能力。

## 产品原则

1. 作者保留最终判断权。
2. AI 只能建议，不能自动污染正史设定。
3. 所有重要生成结果必须结构化、可解析、可保存。
4. 项目上下文必须可追踪来源。
5. 第一版要小，但链路要闭环。

## 从原始文档中采纳的主线

采纳：

- MVP PRD 的清晰边界。
- P0 Implementation Guide 的主闭环。
- 数据库 / 接口 / 前端 / AI 编排文档中的工程化思路。
- AI suggested assets / relations 需要人工确认后入库的原则。
- context_pack 作为 AI 对话上下文载体的设计。

暂不采纳：

- 联邦治理。
- 商业模型信任治理。
- 多 Agent 会议。
- Agent 信誉分。
- 元学习。
- 审计门户。
- Runtime Intervention 全套链路。

## 项目一句话

AgentOS 作品设定工具是一个面向长篇创作的 AI 设定资产工作台，让作者把创意沉淀为可管理、可扩展、可导出的作品知识库。
