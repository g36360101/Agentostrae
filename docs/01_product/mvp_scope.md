# MVP 范围

## MVP 主链路

```text
注册 / 登录
  -> 创建作品项目
  -> 输入创意
  -> 生成高概念候选
  -> 选择高概念
  -> 生成作品核心卡
  -> 生成作品开发案
  -> 抽取设定资产和基础关系
  -> 用户确认保存
  -> 构建项目上下文包
  -> 项目内 AI 对话
  -> 导出作品资料
```

## 第一版必须有

### 项目系统

- 创建项目。
- 编辑项目基础信息。
- 查看项目列表。
- 查看项目详情。

### 高概念生成

- 输入一句或一段创意。
- 生成 3 到 5 个高概念方向。
- 每个方向包含标题、类型、核心卖点、主冲突、主角驱动力、世界差异点、情绪承诺、读者期待、连载潜力和风险提示。
- 用户选择一个作为项目主方向。

### 作品核心卡

- 作品名。
- 类型 / 题材。
- 一句话卖点。
- 读者承诺。
- 世界观概述。
- 主角设定。
- 主角缺口。
- 核心矛盾。
- 反派 / 对立力量。
- 长线悬念。
- 主题表达。
- 目标读者。
- 不可破坏设定。

### 作品开发案

- 作品定位。
- 故事背景。
- 主要角色。
- 主要阵营。
- 世界规则。
- 冲突结构。
- 谜团与伏笔。
- 卷 / 阶段规划。
- 爽点 / 情绪曲线。
- 风险提示。
- 待确认项。

### 设定资产库

资产类型先保持克制：

- `character` 人物
- `location` 地点
- `organization` 组织
- `item` 物品
- `rule` 规则
- `event` 事件
- `mystery` 谜团
- `foreshadowing` 伏笔
- `theme` 主题

### 基础关系

关系类型第一版只需要：

- belongs_to
- allies_with
- opposes
- knows
- hides_from
- caused_by
- depends_on
- located_in
- foreshadows
- reveals

### AI 上下文包

- 选择项目资产。
- 生成 context_pack。
- context_pack 包含本轮任务目标、项目摘要、必须遵守的设定、关键资产、关键关系、隐藏真相、禁用泄露项、风格约束和输出要求。
- AI 对话时引用 context_pack。

### 导出

第一版先导出 Markdown。

默认导出包含：

- 项目简介。
- 作品核心卡。
- 作品开发案。
- 已确认资产库。
- 已确认关系。
- 伏笔与揭示计划。
- 隐藏真相与剧透保护。
- 当前待确认问题。
- AI 上下文摘要。
- 创作过程沉淀摘要。

后续再加：

- DOCX
- PDF
- 分章节写作提示词包

## 第一版不做

- 复杂图谱可视化。
- 自动长篇正文。
- 连载数据分析。
- 读者模拟。
- A/B 测试。
- 多 Agent 投票。
- 自动修复全文。
- 企业治理能力。

## MVP 成功标准

用户可以完成一次完整流程：

1. 输入一个创意。
2. 获得多个方向。
3. 选择方向。
4. 得到一份可读的作品开发案。
5. 保存至少 10 个设定资产。
6. 用这些资产继续与 AI 对话。
7. 导出一份作品设定文档。

内容质量最低标准：

- 至少 3 个高概念候选都能清楚说明卖点、冲突、读者承诺和风险。
- 核心卡能绑定主角、世界和核心矛盾，而不是普通简介。
- 开发案能抽取出至少 10 个有效设定资产。
- 资产必须带叙事功能和来源证据。
- 关系必须带关系类型、证据文本和剧透标记。

如果这条链路稳定，MVP 就成立。
