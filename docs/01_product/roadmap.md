# 开发路线图

## 短期：0 到 4 周

目标：做出可以跑通的作品设定 MVP。

### 第 1 周：项目地基

- 建立前端、后端、数据库基础工程。
- 定义 Project、Asset、Relation、AIJob、ContextPack 的最小模型。
- 实现项目创建、项目列表、项目详情。
- 建立统一 API 返回格式。
- 建立 AI 任务接口，但允许先用 Mock。
- 准备 3 个样例项目的内容种子，用于验证界面和 AI 输出厚度。

### 第 2 周：创意到作品核心卡

- 实现创意输入。
- 实现高概念生成任务。
- 实现高概念选择。
- 实现作品核心卡生成。
- 前端完成项目工作台初版。
- 每个高概念候选必须展示卖点、主冲突、主角驱动力、情绪承诺和风险。
- 核心卡必须展示读者承诺、主角缺口、对立力量和不可破坏设定。

### 第 3 周：开发案与设定资产

- 实现作品开发案生成。
- 实现资产抽取。
- 实现关系抽取。
- 实现用户确认后写入资产库。
- 做基础资产列表和详情页。
- 资产详情补齐叙事功能、剧透等级、来源证据和待补问题。
- 关系记录补齐证据文本、是否隐藏、剧透等级和叙事影响。

### 第 4 周：上下文包与导出

- 实现 context_pack 构建。
- 实现项目内 AI 对话。
- 实现 Markdown 导出。
- 完成 MVP 联调清单。
- 做一轮真实创意样例测试。
- 上下文包必须区分已确认设定、候选内容、隐藏真相和禁止泄露项。
- Markdown 导出必须像作者可使用的作品资料，而不是数据库字段转储。

## 中期：1 到 3 个月

目标：让工具从“能生成”变成“能管理、能迭代”。

### 产品增强

- 关系图谱可视化 v1。
- 资产版本历史。
- 隐藏真相 / 剧透保护。
- 伏笔与揭示计划。
- 分卷规划。
- 章节任务草案。
- 多平台改写提示词。

### AI 增强

- context_pack 评分。
- token 预算与裁剪。
- 隐藏内容阻断。
- 资产冲突检测。
- AI 建议资产入库前审核。
- 开发案质量检查。

### 工程增强

- 完整权限模型。
- 后台任务队列。
- AI 调用日志。
- 成本统计。
- 导出 DOCX。
- E2E 测试。

## 长期：3 到 12 个月

目标：从作品设定工具扩展为 StoryOS 创作操作系统。

### 创作生产链

- 试写启动器。
- 草稿质量报告。
- 修复计划。
- Before / After 对比。
- 读者模拟。
- 真实反馈导入。
- 连载看板。

### 智能协作层

- 图驱动 Agent。
- 工作流控制台。
- 策略实验。
- 项目知识资产库。
- 创作经验图谱。

### 组织级能力

这些能力属于很远的后期，只有在产品已有真实用户后才考虑：

- 多项目治理。
- 审计与合规。
- Agent 信誉分。
- 元学习规则。
- 联邦治理。
- 商业信任模型。

## 路线图判断

短期看闭环，中期看留存，长期看网络效应。

不要在没有真实用户前实现复杂治理系统。治理系统应该从真实使用痛点里长出来，而不是从规划文档里先长满。
