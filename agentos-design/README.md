# AgentOS 界面设计稿

这是一套可直接在浏览器查看、也可作为前端实现依据的产品设计稿。

## 快速查看

打开 `index.html` 可以进入设计总览。

核心交付：

- `boards/web-workspace.html`：Web 端项目工作台，1440px 桌面布局。
- `boards/miniapp-flow.html`：微信小程序三屏流程，覆盖项目状态、快速记录和资产审核。
- `pages/`：完整 Web 核心流程页面，包括项目列表、创意输入、高概念、核心卡、开发案、资产审核和 AI 对话。
- `assets/tiandao-draft-cover.png`：样例作品《天道草稿》的原创封面视觉。
- `previews/`：最终预览图。

## 设计原则

1. Web 是完整创作工作台，适合长时间编辑、比较和批量审核。
2. 小程序只承载移动场景中的轻操作，不复制桌面端复杂编辑器。
3. 界面围绕“作品、设定、关系、过程”四类内容组织。
4. AI 产物默认是候选内容，必须经过用户确认才进入作品正史。
5. 重要页面始终回答三个问题：当前做到哪、有什么待处理、下一步做什么。

## 视觉方向

- 深色中性底色降低长时间创作的视觉干扰。
- 琥珀色用于主操作和创作进度，绿色、蓝色、红色只表达状态。
- 标题使用有编辑气质的衬线字，正文使用系统无衬线字。
- 页面采用密度适中的工作台布局，不使用营销页式的大标题和装饰卡片。
- 卡片圆角不超过 8px，重要信息尽量通过层级、留白和分隔线组织。

## 前端落地建议

- 颜色、间距、圆角和状态语义可迁移到 `packages/ui-tokens`。
- Web 页面按 `apps/web` 的 Next.js 路由拆分。
- 小程序页面按 `apps/miniapp` 的 Taro 页面拆分。
- 两端共享内容模型、状态语义和文案，不强求共享复杂 UI 组件。
